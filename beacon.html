Sociocast Core Beacon API

Purpose

The Sociocast Beacon API is primarily used for entity observations via ad impressions and/or email campaigns. It is useful when you do not have access to JavaScript or other any other server-side languages. The Sociocast Beacon API is an invisible 1Ã—1 pixel image that clients can easily embed in emails or display advertising campaigns. The HTTP Request for the image registers an /entity/observation call in Sociocast's platform. The URL for the image pixel is: 

http://pdata.scastnet.com/pdata.gif
For HTTPS, you can use: 
https://pdata.scastnet.com/pdata.gif
For example, this is how you would load an img tag: 
<img src="http://pdata.scastnet.com/pdata.gif?cid=CLIENT_ID&uuid=UUID&clusid=CLUSTERID1&sc_type=click" height="0px" width="0px"/>

The following URL parameters are acceptable:

String Value Description
eid String The entity ID for the entity.
clid String Your client ID as provided to you by Sociocast. You can also grab this from the Sociocast Console.
url String The URL for the event observation.
clusid Integer One or more clusids to add to the event observation.
Additionally, you can pass in any key=value pair as you would in a general entity observation call.

For ad impression tracking, the request might look like this: 
http://pdata.scastnet.com/pdata.gif?cid=CLIENT_ID&uuid=UUID&url=URL&campaignId=CAMPAIGN_ID&sc_type=view
For ad click tracking, the request might look like this: 
http://pdata.scastnet.com/pdata.gif?cid=CLIENT_ID&uuid=UUID&url=URL&campaignId=CAMPAIGN_ID&sc_type=click
For conversion tracking, the request might look like this: 
http://pdata.scastnet.com/pdata.gif?cid=CLIENT_ID&uuid=UUID&clusid=>CLUSTERID1&clusid=CLUSTERID2&sc_type=click
For email tracking, the request might look like this: 
<img src="http://pdata.scastnet.com/pdata.gif?cid=CLIENT_ID&uuid=UUID&eid.secondary='john%40smith.com'&clusid=CLUSTERID1&sc_type=click" height="0px" width="0px"/>
